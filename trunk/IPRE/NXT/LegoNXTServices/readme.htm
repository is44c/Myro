Lego NXT Services Readme

 

This project contains several services that implement the generic contracts defined in Robotics Common for the Lego NXT.  See Robotics Tutorial 6 for further information on how to implement for your own robot.

 

  • LegoNXTBackupBeep
    • A simple service that subscribes to two motors and tells the brick to play a tone when both motors are turning in reverse.  This service does not implement a Robotics Common generic contract.
  • LegoNXTBattery
    • Implements the generic battery contract in Robotics Common.  Polls for the NXT brick’s battery level every 10 seconds.
  • LegoNXTBumper
    • Implements the generic contact sensor array contract in Robotics Common. 
  • LegoNXTEncoder
    • Implements the generic encoder contract in Robotics Common. 
  • LegoNXTLightSensor
    • Implements the generic light sensor contract in Robotics Common. 
  • LegoNXTMotor
    • Implements the generic motor contract in Robotics Common. 
  • LegoNXTSonar
    • Implements the generic sonar contract in Robotics Common. 
  • LegoNXTSonarAsBumper
    • Implements the generic contact sensor array contract in Robotics Common.  Behind the scenes, it subscribes to a sonar sensor on the NXT brick, and sets its state as ‘pressed’ when the sonar reading is below a threshold.  (See Robotics Tutorial 3 for usage).
  • LegoNXTSound
    • Does not implement a generic contract in Robotics Common.
