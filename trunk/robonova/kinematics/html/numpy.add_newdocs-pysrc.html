 432      of fixed-size items.  An associated data-type-descriptor object 
 433      details the data-type in an array (including byteorder and any 
 434      fields).  An array can be constructed using the numpy.array 
 435      command. Arrays are sequence, mapping and numeric objects. 
 436      More information is available in the numpy module and by looking 
 437      at the methods and attributes of an array.

 440                      offset=0, strides=None, order=None)

 446          all keywords are interpreted. 
 447       The dtype parameter can be any object that can be interpreted

 451       initialized after the __new__ method.

 669  add_newdoc('numpy.core.multiarray', 'ndarray', ('argsort', 
 670      """a.argsort(axis=-1, kind='quicksort', order=None) -> indices 
 671   
 672      Perform an indirect sort along the given axis using the algorithm specified 
 673      by the kind keyword. It returns an array of indices of the same shape as 
 674      'a' that index data along the given axis in sorted order. 
 675   
 676      :Parameters: 
 677   
 678          axis : integer 
 679              Axis to be indirectly sorted. None indicates that the flattened 
 680              array should be used. Default is -1. 
 681   
 682          kind : string 
 683              Sorting algorithm to use. Possible values are 'quicksort', 
 684              'mergesort', or 'heapsort'. Default is 'quicksort'. 
 685   
 686          order : list type or None 
 687              When a is an array with fields defined, this argument specifies 
 688              which fields to compare first, second, etc.  Not all fields need be 
 689              specified. 
 690   
 691      :Returns: 
 692   
 693          indices : integer array 
 694              Array of indices that sort 'a' along the specified axis. 
 695   
 696      :SeeAlso: 
 697   
 698        - lexsort : indirect stable sort with multiple keys 
 699        - sort : inplace sort 
 700   
 701      :Notes: 
 702      ------ 
 703   
 704      The various sorts are characterized by average speed, worst case 
 705      performance, need for work space, and whether they are stable. A stable 
 706      sort keeps items with the same key in the same relative order. The three 
 707      available algorithms have the following properties: 
 708   
 709      |------------------------------------------------------| 
 710      |    kind   | speed |  worst case | work space | stable| 
 711      |------------------------------------------------------| 
 712      |'quicksort'|   1   | O(n^2)      |     0      |   no  | 
 713      |'mergesort'|   2   | O(n*log(n)) |    ~n/2    |   yes | 
 714      |'heapsort' |   3   | O(n*log(n)) |     0      |   no  | 
 715      |------------------------------------------------------| 
 716
