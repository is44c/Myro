newbyteorder(...)

 
self.newbyteorder(<endian>) returns a copy of the dtype object with altered byteorders. If <endian> is not given all byteorders are swapped. Otherwise endian can be '>', '<', or '=' to force a particular byteorder. Data-types in all fields are also updated in the new dtype object.
